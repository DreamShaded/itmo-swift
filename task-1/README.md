# Симуляция жизни зоопарка (Swift)

Консольная ООП-программа: животные едят, стареют, размножаются и умирают по
случайным событиям. Каждое животное — отдельный объект; виды — классы-наследники
базового `Animal`. Симуляция идёт по итерациям («дням») и печатает журнал событий
и статистику.

Решение по [`task.md`](./task.md). Реализован **обязательный минимум**.
Звёздные (бонусные) задания — не сделаны намеренно; ниже расписано, как добавить
их самостоятельно.

## Как запустить

Нужен Swift 5.9+ (см. [`../SWIFT-SETUP.md`](../SWIFT-SETUP.md), если Swift ещё не установлен).

```bash
cd task-1
swift run                                    # SwiftPM соберёт и запустит
# либо без SwiftPM, напрямую компилятором:
swiftc Sources/Zoo/*.swift -o zoo && ./zoo
```

> В этом окружении (CachyOS, тулчейн в `~/swift-toolchain`) перед запуском нужно:
> ```bash
> export PATH="$HOME/swift-toolchain/usr/bin:$PATH"
> export LD_LIBRARY_PATH="$HOME/swift-libs:/usr/lib"
> ```

## Что где лежит (карта проекта)

```
task-1/
├── Package.swift                  # манифест SwiftPM: исполняемый таргет "Zoo"
├── README.md                      # этот файл
├── .gitignore                     # игнорирует .build/
└── Sources/Zoo/
    ├── SeededGenerator.swift      # генератор случайных чисел (детерминированный)
    ├── Animal.swift               # БАЗОВЫЙ класс: состояние + поведение животного
    ├── Species.swift              # виды-наследники: Wolf, Bear, Cat, Fox, Chicken
    ├── Zoo.swift                  # менеджер симуляции: итерации, события, статистика
    └── main.swift                 # точка входа: заселение зоопарка и запуск
```

**Куда смотреть в первую очередь:** `main.swift` (как всё запускается) →
`Animal.swift` (общее поведение) → `Species.swift` (чем отличаются виды) →
`Zoo.swift` (что происходит за одну итерацию).

### Иерархия классов

```
Animal  (Animal.swift)
 ├── Wolf      живёт долго,      ест много,     размножается редко
 ├── Bear      живёт очень долго, ест много,     размножается редко
 ├── Cat       средняя жизнь,    ест умеренно,   средняя плодовитость
 ├── Fox       короткая жизнь,   ест умеренно,   средняя плодовитость
 └── Chicken   живёт мало,       ест мало,       размножается часто
```

### За что отвечает каждый файл

| Файл | Содержимое | Ключевое |
| --- | --- | --- |
| `SeededGenerator.swift` | ГСЧ на SplitMix64, реализует `RandomNumberGenerator` | `percent()` → 1...100, `happens(p)` → бросок на вероятность `p%` |
| `Animal.swift` | Базовый класс: поля состояния + методы поведения | `eat()`, `starve()`, `growOlder()`, `isDead()`, `tryReproduce()`, `die()`, `printInfo()`, `makeOffspring()` |
| `Species.swift` | 5 наследников, каждый задаёт свои числа и переопределяет `makeOffspring` | параметры вида в `init` |
| `Zoo.swift` | Логика симуляции и весь вывод в консоль | `addAnimal`, `runIteration`, `runSimulation`, статистика |
| `main.swift` | Создаёт зоопарк, добавляет животных, запускает | здесь меняются `seed`, число дней, состав |

## Как это соответствует ТЗ (обязательный минимум)

| Требование ТЗ | Где |
| --- | --- |
| Базовый класс `Animal` с полями (name, species, age, maxAge, hunger, isAlive, reproductionChance, foodNeed) | `Animal.swift` |
| Методы `eat / growOlder / tryReproduce / die / isDead / printInfo` | `Animal.swift` |
| ≥4 класса-наследника, отличающихся параметрами | `Species.swift` (5 видов) |
| Список/массив объектов животных | `Zoo.animals: [Animal]` |
| Генератор случайных чисел | `SeededGenerator.swift` |
| Цикл симуляции по итерациям | `Zoo.runIteration()` / `runSimulation(iterations:)` |
| Журнал событий в консоль | печать внутри `runIteration()` |
| Финальная статистика | `Zoo.printFinalStatistics()` |
| `addAnimal / removeDeadAnimals / runIteration / runSimulation / printStatistics` | `Zoo.swift` |

**Полиморфизм без `if`/`switch` по виду:** `Zoo` хранит `[Animal]` и вызывает
переопределённый `makeOffspring(name:)` — нужный вид потомка определяется
динамически самим объектом.

**Воспроизводимость:** ГСЧ инициализируется фиксированным `seed` (`main.swift`),
поэтому каждый запуск даёт один и тот же ход событий. Меняйте `seed` — меняется
сценарий.

### Модель голода (как работает «поел / не нашёл еду»)

- `hunger` стартует с 0; `0` = сыт.
- День без еды: `hunger += foodNeed`. День с едой: `hunger -= foodNeed` (не ниже 0).
- Смерть от голода: `hunger >= hungerLimit`. Смерть от старости: `age > maxAge`.

Все числа задаются в `Species.swift` — балансировать поведение видов можно прямо там.

## Что можно докрутить (звёздные задания)

Каждое — небольшое и локальное изменение. Ориентиры, с чего начать:

### ★ 1. Разные типы питания (хищник / травоядный / всеядный)
- В `Animal.swift` добавьте `enum Diet { case carnivore, herbivore, omnivore }` и поле `let diet: Diet`.
- Прокиньте `diet` через `init` базового класса и задайте в каждом виде (`Species.swift`).
- В `Zoo.swift` шанс «найти еду» можно сделать зависимым от `diet`.

### ★ 2. Пищевая цепочка / охота
- В `Zoo.runIteration()` для хищника добавьте случайную попытку охоты: выбрать
  случайную жертву подходящего вида из `animals`, при удаче — жертва получает
  смерть (`die()`), хищник `eat()`.
- Кого на кого: волк → курица/кролик, лиса → курица, медведь → почти все, кошка → мелкие.
  Удобно завести метод `func canHunt(_ prey: Animal) -> Bool` на `Animal`/видах.
- **Важно (по ТЗ):** охота должна быть случайной (`rng.happens(...)`).

### ★ 3. Пол животного
- В `Animal.swift` добавьте `enum Sex { case male, female }` и поле `let sex: Sex`
  (генерировать случайно в `init`). Не путать с `Gender` — тот лишь для грамматики глаголов.
- В `Zoo`: размножение разрешать, только если есть **двое взрослых одного вида разного пола**.
  То есть `tryReproduce` перенести в `Zoo` как проверку по парам, а не по одному животному.

### ★ 4. Здоровье
- В `Animal.swift` добавьте `var health: Int` и `let maxHealth`.
- Уменьшать `health` при голоде/старости/случайной болезни (новый бросок ГСЧ в `Zoo`).
- В `isDead()` учесть `health <= 0` как ещё одну причину смерти (дополните `deathReason()`).

### ★ 5. Имена потомков из списка
- Сейчас потомки зовутся `#1`, `#2` (счётчик в `Zoo.nextOffspringName`).
- Замените на выбор из заранее подготовленного массива имён (например, по виду):
  храните `[String: [String]]` и берите случайное/по порядку имя в `nextOffspringName`.

### ★ 6. Настройки симуляции
- Сейчас параметры заданы в `main.swift` (`seed`, `foodChance`, `maxPopulation`, число дней).
- Считайте их из аргументов командной строки через `CommandLine.arguments`
  (или интерактивно через `readLine()`), передайте в `Zoo(...)` и `runSimulation(...)`.
  Пример: `swift run Zoo --days 30 --food 70 --max 50`.

После любого изменения проверяйте сборку: `swift run` (или
`swiftc Sources/Zoo/*.swift -o zoo && ./zoo`). Файлы стоит держать в пределах
~200 строк — при росте выносите новый функционал в отдельные файлы (например,
`Diet.swift`, `Hunting.swift`).

...

Статистика:
Всего животных: 6

  Волк: 1
  Кошка: 1
  Курица: 2

В конце симуляции печатается блок **«Итоги симуляции»**: сколько дней прошло,
сколько животных осталось, разбивка по видам и финальное состояние каждого
выжившего (`printInfo`).
