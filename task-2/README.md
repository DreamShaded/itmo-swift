## Ход работы (методология)

ТЗ ориентировано на ручной перехват трафика телефона через Charles/Proxyman.
В этом окружении **нет телефона и прокси**, но **бэкенд-стенд доступен напрямую**,
поэтому работа велась двумя путями, которые в сумме покрывают задание:

1. **Прямые HTTP-запросы через `curl`** — ТЗ явно разрешает replay/Postman/curl как
   замену перехвату. Так выполнены разведка API, все эксперименты с изменением
   запросов и проверка серверного состояния.
2. **Статический анализ APK** — вместо перехвата трафика приложения (Часть 2)
   эндпоинты и структура запросов извлечены прямо из кода приложения.

### Шаг 1. Разведка API (Часть 1)

Swagger по адресам `/api-docs`/`/swagger.json` отдаёт 404. OpenAPI-спека встроена
в JS Swagger UI — вытащили её оттуда:

```bash
# спека лежит внутри swagger-ui-init.js как объект swaggerDoc
curl -s http://82.39.214.8:3001/api-docs-ui/swagger-ui-init.js -o swagger-init.js
# затем извлечение JSON по балансу скобок (см. ниже про инструменты) → 23 эндпоинта
```

Результат: 23 метода на двух поверхностях — `/api/*` (используется приложением) и
легаси `/cats/*` (теневой API). Таблица — в `report.md` §2.

### Шаг 2. Карта действий приложения через анализ APK (Часть 2)

`jadx`/`apktool` недоступны, поэтому APK (это zip) распакован и просканирован `strings`:

```bash
mkdir apk && cd apk
unzip -oq ../Meowle-lab.apk
# базовый URL и эндпоинты, которые реально вызывает приложение:
strings -n 6 classes*.dex | grep -iE "82\.39\.214\.8|/api/|/cats/"
# имя multipart-поля загрузки, DTO-классы:
strings -n 4 classes*.dex | grep -iE "Multipart|AddCatDto|catDescription|gender"
```

Вывод: приложение использует **только `/api/*`**; легаси `/cats/*` в коде нет →
это и есть «методы, которых нет в мобильном сценарии». Карта — в `report.md` §3.

### Шаг 3. Эксперименты с изменением запросов (Часть 3)

Все эксперименты собраны в одну воспроизводимую bash-обвязку, которая логирует
каждый запрос и ответ в Markdown. Покрыто: IDOR, накрутка лайков, правка чужого
описания (с восстановлением), дубликаты, невалидные значения, теневой API,
несуществующий `catId`, варианты загрузки файлов. Подробности — `report.md` §4,
сырой лог — `evidence/log.md`.

### Шаг 4. Изменение ответов (Часть 4)

Подмена ответов (Charles Map/Rewrite) физически не выполнялась (нет прокси+UI), но
её суть доказана через API: правки **ответов** видны только клиенту, а источник
истины — сервер. Это подтверждено повторными GET-запросами. См. `report.md` §5.

### Шаг 5. Анализ безопасности и баг-репорты (Части 5–6)

На основе собранных доказательств составлены **8 находок по безопасности** и **7 баг-репортов**
в требуемом формате (Заголовок / Серьёзность / Шаги / Доказательство / Рекомендация). См.
`report.md` §6–7.

## Ключевые находки (кратко)

- 🔴 **Критично** — авторизации нет вообще; «вход» по имени+телефону чисто клиентский.
- 🟠 **IDOR / анонимное удаление** — лайк/правка/удаление любого объекта по `id`.
- 🟠 **Утечка ошибки PostgreSQL** — `DELETE /cats/{id}/dislike` отдаёт схему БД, таблицу, constraint.
- 🟠 **Загрузка файлов** — текстовый файл принят как `image/png` и отдаётся как картинка.
- 🟡 **Слабая валидация** — невалидный `gender` молча заменяется на `unisex`.

Полный список с доказательствами — `report.md`.

## Как воспроизвести

> ⚠️ Только на учебном стенде Meowle и в рамках ТЗ. Без массового сканирования,
> брутфорса и DoS. Чужие данные не портить (а если меняли для доказательства —
> восстанавливать, как сделано здесь).

Примеры отдельных проверок:

```bash
B=http://82.39.214.8:3001

# рейтинг / поиск (безопасные GET/POST)
curl -s "$B/api/likes/cats/rating"
curl -s -X POST "$B/api/core/cats/search" -H 'Content-Type: application/json' -d '{"name":""}'

# IDOR-проверка: лайк произвольного кота без авторизации
curl -s -X POST "$B/api/likes/cats/18/likes" -H 'Content-Type: application/json' -d '{"like":true,"dislike":false}'

# утечка ошибки БД на легаси-эндпоинте
curl -s -X DELETE "$B/cats/18/dislike"

# загрузка не-изображения как image/png
printf 'not an image' > fake.png
curl -s -X POST "$B/api/photos/cats/<свой_id>/upload" -F 'file=@fake.png;type=image/png'
```

Полный набор команд с фактическими ответами — в `evidence/log.md`.

## Что осталось для ручного выполнения (нужен телефон + прокси)

Эти пункты требуют физического Android-устройства и Charles/Proxyman и здесь не
делались:

1. **Часть 2 — скриншоты реального перехвата** трафика приложения (карта действий
   реконструирована из APK, но без живых скриншотов прокси).
2. **Часть 4 — скриншоты «до/после»** подмены ответов в UI приложения (суть доказана
   через API, но без визуальных скриншотов экрана).

Как доделать:
- Настроить прокси по инструкции из `Лаба_ТЗ.md` (раздел «Настройка прокси»).
- Выполнить в приложении 8 действий из Части 2, сохранить пары запрос/ответ из прокси.
- В Charles применить Map Local/Rewrite к 4 ответам из Части 4, сделать скриншоты до/после.
- Добавить скриншоты в `report.md` (или приложением) — текстовая часть уже готова.
